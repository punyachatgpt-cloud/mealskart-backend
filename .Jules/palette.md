## 2025-05-22 - [Scope Management for Micro-UX]
**Learning:** For "Palette" tasks with a strict 50-line limit, bundling multiple accessibility fixes (modal focus, button states, card navigation) into one PR is likely to fail code review due to complexity and line count. It's better to pick one high-impact area and execute it perfectly.
**Action:** Focus on one specific component or interaction per task. If multiple improvements are needed, prioritize based on user impact (e.g., keyboard navigability of primary content).
